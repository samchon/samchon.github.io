Class SetContainer<T>

An abstract set.

SetContainers are containers that store elements allowing fast retrieval of individual elements based on their value.

In an SetContainer, the value of an element is at the same time its key, used to identify it. Keys are immutable, therefore, the elements in an SetContainer cannot be modified once in the container - they can be inserted and removed, though.

SetContainer stores elements, keeps sequence and enables indexing by inserting elements into a List and registering iterators of the list container to an index table like {@link RBTree tree} or hash-table.

Container properties

Associative
Elements in associative containers are referenced by their key and not by their absolute position in the container.
Set
The value of an element is also the key used to identify it.

end

  • end(): SetIterator<T>

  • Implementation of IContainer.end

    Overwrites Container.end

    • Defined in base/container/SetContainer.ts:184

    Return iterator to end.

    Returns an iterator referring to the past-the-end element in the container.

    The past-the-end element is the theoretical element that would follow the last element in the container. It does not point to any element, and thus shall not be dereferenced.

    Because the ranges used by functions of the Container do not include the element reference by their closing iterator, this function is often used in combination with IContainer.begin to specify a range including all the elements in the container.

    Note

    Returned iterator from IContainer.end does not refer any element. Trying to accessing element by the iterator will cause throwing exception (OutOfRange).

    If the container is empty, this function returns the same as Container.begin.

    Returns SetIterator<T>

    An iterator to the end element in the container.

  • handle_erase(item: SetIterator<T>): void
    • Defined in base/container/SetContainer.ts:459

    Abstract method handling deletion for indexing.

    This method, handle_insert is designed to unregister the item to somewhere storing those iterators for indexing, fast accessment and retrievalance.

    When erase is called with item, an iterator positioning somewhere place to be deleted, is memorized and shifted to this method handle_erase after the deletion process is terminated.

    If the derived one is {@link RBTree tree-based} like TreeSet, the item will be unregistered from the tree as a tree node item. Else if the derived one is hash-based like HashSet, the item will be unregistered from the hash bucket.

  • handle_insert(item: SetIterator<T>): void
    • Defined in base/container/SetContainer.ts:440

    Abstract method handling insertion for indexing.

    This method, handle_insert is designed to register the item to somewhere storing those iterators for indexing, fast accessment and retrievalance.

    When insert is called, a new element will be inserted into the list container and a new iterator item, pointing the element, will be created and the newly created iterator item will be shifted into this method handle_insert after the insertion.

    If the derived one is {@link RBTree tree-based} like TreeSet, the item will be registered into the tree as a tree node item. Else if the derived one is hash-based like HashSet, the item will be registered into the hash bucket.
