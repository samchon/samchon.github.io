Detailed Description

스케쥴러. 시공 일정을 최적화 합니다.

시공일정 최적화는 일종의 TSP 알고리즘(TSP, Traveling Salesman Problem)에 해당하는 영역으로써, 이론적으로는 순열(N!, Factorial case) 을 통하여 해결할 수 있습니다. 하지만, 이는 어디까지나 이론적인 얘기일 뿐, N!이라는 수는 워낙 거대한 수인지라 (N이 15를 넘으면 연산에 1시간이 걸리고, 20을 넘으면 215년이 걸린다.) 순열(N!)은 현실적으로 도입할 수 없는 알고리즘입니다.

  • 참고자료: https://en.wikipedia.org/wiki/Travelling_salesman_problem

때문에 실제 시공일정에 대한 완벽한 답을 구하는 것이 아니라, 최적의 근사치를 구해야 합니다. 저는 이 시공일정의 최적화에 k-means 알고리즘과 유전자 알고리즘을 도입하여 해결하였습니다.

  • k-means 알고리즘: https://en.wikipedia.org/wiki/K-means_clustering
  • 유전자 알고리즘: https://en.wikipedia.org/wiki/Genetic_algorithm

void Scheduler::optimizeByByKMeans ( )
private

K-means 알고리즘을 통한 최적화

K-means 알고리즘과 Factorial case generator를 다음과 같이 적용합니다.

  1. K-means 알고리즘을 적용하여 전체 지점을 9개의 구획으로 나눕니다.
  2. 각 구획의 중앙점 간에 이동거리를 Factorial case generator(N! 경우의 수 생성기)를 사용, 구획간 재배치를 함으로써 최적화합니다.
  3. 각 구획 내 지점의 수가 9개 이하가 될 때까지, 각 구획에 대해서도 1, 2의 과정을 반복합니다. 즉, 각 구획 내의 좌표가 9개 이하가 될 때까지, 각 구획을 다시 부분구획으로 쪼개어 나가는 것입니다.
  4. 각 구획 내 좌표에 대하여, 다시 Factorial case generator를 사용하여 순서를 재배열합니다.
Note

유전자 알고리즘의 소요 시간은, 진화 세대수나 세대 당 인구수, 돌연변이율 및 토너먼트 횟수 등 (이하 유전자 알고리즘 파라미터) 에 크게 밀접합니다. 유전자 알고리즘 파라미터 값은 초기 데이터셋이 최적에 얼마나 가깝냐에 따라 다르게 결정하게 됩니다. 초기 데이터 셋이 최적에 가까울수록 유전자 알고리즘 파라미터에 작은 값을 줄 수 있습니다.

때문에, ConstructionScheduler에서의 K-means 알고리즘과 Factorial case generator 용도는 바로, 이 유전자 알고리즘 파라미터를 작게 가져가기 위한 사전 정지작업입니다.

Warning

K-means 알고리즘과 Factorial case generator는, 오로지 이동거리만을 최적화할 뿐, 제약조건을 고려하지 않습니다.
